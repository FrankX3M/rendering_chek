# 🔍 HTML Diff Analysis Tools

Набор инструментов для визуального анализа и сравнения HTML-страниц с автоматической подсветкой различий между версиями для ботов и пользователей.

## 📦 Состав пакета

### ⭐ `highlight_diff.py` - ГЛАВНЫЙ ИНСТРУМЕНТ

**Это именно то, что вам нужно!**

Автоматически:
- ✅ Берёт HTML для ботов и пользователей
- ✅ Находит различия (игнорирует динамические CSS)
- ✅ Подсвечивает прямо на странице:
  - 🟢 **Зелёным** — элементы только для ботов
  - 🔴 **Красным** — элементы только для пользователей
- ✅ Создаёт интерактивное side-by-side сравнение

### `analyze_single_with_highlight.py` - Анализ одного файла

Быстрый анализ структуры любого HTML файла с интерактивной визуализацией:
- Статистика по элементам
- Поиск и фильтрация
- Проверка SSR/CSR
- Обнаружение фреймворков

### `analyze_html_diff.py` - Детальное сравнение

Подробный текстовый отчёт о различиях между двумя HTML файлами.

## 🚀 Быстрый старт

### Сценарий 1: Визуальное сравнение с подсветкой (РЕКОМЕНДУЕТСЯ)


**Что вы увидите:**
- Side-by-side сравнение с синхронизацией скролла
- 🟢 Зелёная подсветка элементов только для ботов
- 🔴 Красная подсветка элементов только для пользователей

- CSS селекторы для копирования

### Сценарий 2: Анализ одного файла

```bash

```

**Что покажет:**
- Общую статистику (элементы, теги, ссылки)
- Интерактивный поиск по элементам
- Проверку SSR (заполнен ли root)
- Топ-20 самых частых тегов
- Предпросмотр страницы

### Сценарий 3: Детальное сравнение


## 🔧 Продвинутое использование

### Настройка игнорирования классов

Если нужно игнорировать дополнительные динамические классы, отредактируйте в скрипте:

```python
ignored_class_patterns = [
    r'^css-[a-z0-9]+$',      # CSS-in-JS
    r'^sc-[a-zA-Z0-9]+-[a-zA-Z0-9]+$',  # Styled-components
    r'^_[a-z0-9]+$',         # CSS Modules
    r'^[a-zA-Z]+-[0-9]+$',   # Tailwind JIT
    r'.*-\d{6,}$',           # Hash классы
    r'^your-pattern-here$',  # ← Добавьте свой паттерн
]
```

### Экспорт результатов в JSON

```python
# Добавьте в конец скрипта highlight_diff.py
import json
with open('diff.json', 'w', encoding='utf-8') as f:
    json.dump({
        'only_bot': [e['selector'] for e in comparison['only_in_bot']],
        'only_user': [e['selector'] for e in comparison['only_in_user']],
    }, f, indent=2, ensure_ascii=False)
```

### Batch обработка нескольких страниц

```bash
#!/bin/bash
# process_all.sh

for page in home category product; do
    echo "Processing $page..."
    python highlight_diff.py \
        "bot_${page}.html" \
        "user_${page}.html" \
        -o "comparison_${page}.html"
done

echo "✅ All done! Open comparison_*.html files"
```

## 📋 Интерпретация результатов

### ✅ Зелёный сценарий (всё хорошо)

```
🟢 Только для ботов: 0-5 элементов
🔴 Только для пользователей: 0-10 элементов
✅ Общие элементы: 95%+
```

**Вывод:** SSR работает отлично

**Примеры нормальных различий:**
- 🔴 Реклама (Google Ads, баннеры)
- 🔴 Трекеры (Google Analytics, Яндекс.Метрика)
- 🔴 Чаты (Jivosite, Carrot quest)
- 🔴 Личный кабинет (профиль, избранное)

### ⚠️ Жёлтый сценарий (требуется проверка)

```
🟢 Только для ботов: 5-20 элементов
🔴 Только для пользователей: 10-50 элементов
✅ Общие элементы: 80-95%
```

**Вывод:** Гибридный рендеринг, нужна проверка

**Проверьте:**
- Что именно в 🔴? Если контент → проблема
- Есть ли важные ссылки в 🔴? → проблема
- Все ли H1-H3 одинаковые? → критично

### ❌ Красный сценарий (критично!)

```
🟢 Только для ботов: 20+ элементов
🔴 Только для пользователей: 50+ элементов
✅ Общие элементы: <80%
```

**Вывод:** Серьёзные проблемы с SSR

**Критичные проблемы:**
- 🔴 Весь контент (товары, статьи) → CSR, Google не видит
- 🔴 Навигация и ссылки → Google не может краулить
- 🟢 Разные H1/Title → Cloaking, бан от Google!
- Пустой `#__next` или `#root` → SSR сломан

## 🐛 Решение проблем

### Проблема: "Слишком много различий"

**Причина:** Динамические классы не фильтруются

**Решение:**
1. Проверьте в `comparison_viewer.html` вкладки с элементами
2. Посмотрите на классы элементов
3. Добавьте паттерны в `ignored_class_patterns`

### Проблема: "Различий нет, но визуально разные"

**Причина:** Различия в JavaScript, не в HTML структуре

**Решение:**
- Различия могут быть в стилях (CSS)
- Различия могут быть в JS-логике
- Используйте вкладку "Визуальное сравнение" для проверки

### Проблема: "Не открывается comparison_viewer.html"

**Причина:** Относительные пути к highlighted файлам

**Решение:**
```bash
# Убедитесь, что все файлы в одной директории:
ls -la *.html

# Должны быть:
# comparison_viewer.html
# *_highlighted.html (2 файла)
```

## 📝 FAQ

**Q: Можно ли игнорировать определённые элементы?**

A: Да, добавьте проверку в функцию `extract_structure()`:

```python
if element.get('class') and 'ignore-me' in element.get('class'):
    return  # Пропускаем этот элемент
```

**Q: Как найти элемент на реальной странице?**

A: Скопируйте CSS селектор из отчёта и вставьте в DevTools → Console:

```javascript
document.querySelector('your-selector-here')
```

**Q: Можно ли сравнить больше 2 версий?**

A: Сейчас нет, но можно запустить несколько раз:

```bash
python highlight_diff.py v1.html v2.html -o diff_1_2.html
python highlight_diff.py v2.html v3.html -o diff_2_3.html
```

**Q: Работает ли с Next.js / Nuxt / Gatsby?**

A: ✅ Да! Работает с любыми фреймворками и SSR решениями

**Q: Нужен ли Selenium / Playwright?**

A: ❌ Нет! Скрипты работают с готовыми HTML файлами
