# Инструкция по работе с SSR / Cloaking Checker

## Что делает программа

Программа сравнивает HTML-страницу в двух вариантах:

- **BOT** — как её видит поисковый робот (Googlebot и др.). Playwright автоматически загружает страницу с нужным User-Agent.
- **USER** — как её видит обычный пользователь в браузере. HTML вы копируете и вставляете вручную.

По результатам сравнения строится HTML-отчёт с тремя вкладками: **Analysis**, **Elements**, **Prototype**.

Если контент для бота и пользователя одинаковый — это полноценный SSR. Если разный — это cloaking или Dynamic Rendering.

---

## Установка

### 1. Требования

- Python 3.10 или новее
- pip

### 2. Установка зависимостей

```bash
pip install playwright python-dotenv beautifulsoup4
playwright install chromium
```

### 3. Файл `.env` (необязательно)

Создайте `.env` в папке со скриптом, если хотите задать настройки по умолчанию:

```env
BOT_MODE=google_smartphone
TARGET_URL=https://example.com
ANTIBOT_HEADER_NAME=X-Custom-Header
ANTIBOT_HEADER_VALUE=секретное_значение
GOTO_TIMEOUT=60000
SELECTOR_TIMEOUT=15000
DYNAMIC_EXTRA_SLEEP=3
```

Все параметры необязательны — можно менять прямо в меню программы.

---

## Запуск

```bash
python checker.py
```

Можно сразу передать URL аргументом:

```bash
python checker.py https://example.com/product/123
```

## Пошаговый процесс проверки

### Шаг 0 — введите URL

Выберите пункт **2. Изменить URL** и введите адрес страницы, которую хотите проверить. Можно вводить без `https://` — программа добавит протокол сама.

Уже проверенные URL сохраняются в истории (до 50 штук) — можно выбрать по номеру.

---

### Шаг 1 — загрузка как БОТ (автоматически)

Выберите пункт **1. Запустить проверку**.

Playwright автоматически откроет страницу в headless-браузере с User-Agent поискового робота. Вам ничего делать не нужно. Программа:

1. Открывает страницу с таймаутом до 60 секунд.
2. Ждёт появления динамических элементов (цена, кнопка корзины и др.).
3. Делает паузу 3 секунды для завершения JS.
4. Ждёт состояния «network idle».
5. Считывает итоговый DOM.

В консоли вы увидите подробный лог загрузки.

---

### Шаг 2 — получение HTML как ПОЛЬЗОВАТЕЛЬ (вручную)

После завершения загрузки бота программа попросит вас вставить HTML страницы глазами обычного пользователя.

**Откройте ту же страницу в обычном браузере** (Chrome, Firefox, Edge) и скопируйте HTML одним из способов:

---

#### ✅ Способ 1 — «Просмотр кода страницы» (рекомендуется, быстрее)

Это исходный HTML до выполнения JavaScript — подходит для проверки SSR.

1. Откройте нужную страницу в браузере.
2. **Кликните правой кнопкой мыши** в любом месте страницы (не на картинке и не на ссылке).
3. В контекстном меню выберите **«Просмотр кода страницы»** (или «View Page Source» в английском браузере).
4. Откроется новая вкладка с HTML-кодом.
5. Нажмите **Ctrl+A** — выделить всё.
6. Нажмите **Ctrl+C** — скопировать.

---

#### Способ 2 — Инструменты разработчика (DOM после JS)

Этот вариант даёт HTML уже после выполнения JavaScript — подходит для анализа гидратации.

1. Откройте нужную страницу в браузере.
2. Нажмите **F12** или **Ctrl+Shift+I** — откроются DevTools.
3. Перейдите на вкладку **Elements** (Элементы).
4. В дереве найдите тег `<html>` в самом верху.
5. Кликните на него **правой кнопкой мыши**.
6. Выберите **Copy → Copy outerHTML**.

---

После копирования вернитесь в консоль с программой и **вставьте HTML** (Ctrl+V или правой кнопкой → Вставить).

Для завершения ввода нажмите **Enter на пустой строке** (просто Enter без текста).

> ⚠️ Важно: вставляйте HTML сразу после копирования, не открывая другие страницы и не копируя ничего другого, иначе буфер обмена перезапишется.

---

### Шаг 3 — сравнение (автоматически)

Программа автоматически сравнивает два HTML и выводит в консоль:

- Схожесть текста в процентах
- Совпадение хешей контента
- Разницу в количестве H1, H2, P, IMG, A
- Динамические поля: цена, рейтинг, отзывы, стикеры, кнопка корзины, промо, лояльность
- Итоговый вывод: SSR / Гибридный SSR / Cloaking

---

### Шаг 4 — генерация отчёта (автоматически)

Программа создаёт файл отчёта в текущей папке с именем вида:

```
report_20250302_143827_https___example_com.html
```

Формат имени: `report_ГГГГММДД_ЧЧММСС_URL.html` — каждая проверка создаёт отдельный файл, файлы не перезаписываются.

После генерации программа спросит: **«Открыть отчёт в браузере прямо сейчас?»**

- Нажмите **Y** (или просто Enter) — сервер запустится автоматически и отчёт откроется в браузере.
- Нажмите **N** — откроете позже через пункт меню 5.

## Просмотр отчёта (пункт меню 5)

Отчёт — обычный HTML-файл, но из-за CORS его нельзя просто открыть двойным кликом. Нужен локальный веб-сервер.

Выберите **5. 🖥 Web-сервер / отчёты** в главном меню.

Подменю:

```
  ✅ Сервер работает  →  http://localhost:8000/

  📄 Доступные отчёты:
     1. report_20250302_143827_...html  (248 KB, 02.03 14:38)  ◀ последний
     2. report_20250301_110512_...html  (231 KB, 01.03 11:05)

  1. 🔗 Открыть отчёт в браузере (выбрать)
  2. 🔗 Открыть последний отчёт
  3. ⛔ Остановить сервер
```

- Сервер работает в фоне — можно продолжать проверки не останавливая его.
- При выходе из программы (пункт 0) сервер останавливается автоматически.

## Отчёт — описание вкладок

### 📊 Analysis

Сводная статистика по двум версиям страницы:

| Показатель | Описание |
|---|---|
| Title | Заголовок `<title>` |
| Description | Мета-описание |
| File Size | Размер HTML |
| Elements | Количество всех HTML-элементов |
| Scripts / Images / Links | Счётчики |
| CSS blocks | Блоки CSS-in-JS (emotion) |

Также показывается состояние динамических полей: цена, лояльность, промо, остаток, рейтинг, отзывы, стикеры, наличие кнопки «В корзину».

### 🔬 Elements

Детальное сравнение HTML-элементов по группам:

- **Теги** — таблица всех тегов с количеством у бота и пользователя, разница подсвечена.
- **Диаграмма** — визуальные полосы для сравнения.
- **data-qa** — атрибуты `data-qa`: общие, только у бота, только у пользователя.
- **CSS** — топ-30 CSS-классов с разницей.
- **Прочее** — сводная таблица всех метрик + нестандартные теги.

### 📱 Prototype

Визуальная схема страницы в виде макета телефона. Показывает структуру блоков:

- Одинаковые блоки — серые.
- Блоки только у бота — зелёная пунктирная рамка.
- Блоки только у пользователя — красная пунктирная рамка.

Над макетами показывается счётчик отличий.

## Настройки (пункт меню 4)

| Параметр | Описание | По умолчанию |
|---|---|---|
| Bot Mode | User-Agent бота | `google_smartphone` |
| AntiBot Header | Специальный заголовок для обхода антибот-защиты | пусто |
| GOTO timeout | Таймаут загрузки страницы (мс) | 60000 |
| Selector timeout | Таймаут ожидания динамических элементов (мс) | 15000 |
| Extra sleep | Дополнительная пауза после загрузки (сек) | 3 |

### Доступные режимы бота (Bot Mode)

| Режим | Описание |
|---|---|
| `google_smartphone` | Googlebot — мобильный (по умолчанию) |
| `google_desktop` | Googlebot — десктоп |
| `chrome_desktop` | Обычный Chrome — десктоп |
| `chrome_mobile` | Обычный Chrome — мобильный |
| `fake_bot` | AhrefsBot (сторонний бот) |

### AntiBot Header

Если сайт использует специальный HTTP-заголовок для разрешения доступа ботам (например, внутренний bypass-токен), укажите его имя и значение. Программа будет передавать его при каждом запросе.

## Интерпретация результатов

| Результат | Значение |
|---|---|
| ✅ Схожесть >95%, разница текста <5% | Полноценный SSR — контент одинаковый |
| ⚠️ Схожесть 70–95% | Гибридный SSR — возможна персонализация или A/B-тест |
| ❌ Схожесть <70% | Cloaking или Dynamic Rendering — разный контент для ботов |
| ⚠️ Root бота заполнен, у пользователя пуст | SSR только для ботов — классический cloaking |

## Типичные ошибки

**`playwright не установлен`**
```bash
pip install playwright
playwright install chromium
```

**`Порт 8000 занят`**
Программа автоматически попробует порт 8001. Или вручную выберите другой порт в подменю сервера.

**Вставленный HTML не принимается**
Убедитесь что после последней строки HTML нажали **Enter на пустой строке** (Enter без текста). Не нажимайте Ctrl+C — это отменит ввод.

**Отчёт не открывается двойным кликом**
Это нормально из-за CORS. Используйте пункт меню **5. Web-сервер** или запустите вручную:
```bash
python -m http.server 8000
```
Затем откройте `http://localhost:8000/report_....html`.
